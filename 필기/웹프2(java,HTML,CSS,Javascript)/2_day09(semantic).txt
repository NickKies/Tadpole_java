시멘틱태그의 기본적인 구성
시맨틱은 "의미, 의미론적인" 이라는 뜻입니다. html5에 동ㅂ된 시멘틱태그는 개발자와 브라우저에게 의미있는 태그를 제공하게 됩니다.
예) 
<div> : non-semantic 태그, 안에 들어갈 으미를 크게 유추하지 못합니다.
<article> : semantic 태그, 특정형태의 글이 포함될 것이라는 유추가 가능합니다.

header : 상단, 헤더를 의미 
nav : 메뉴, 내비게이션을 의미
aside : 사이드에 위치하는 공간을 의미
section :  여러 중심 내용을 감싸는 공간을 의미
article :  글자가 많이 들어간 부분을 의미
footer : 하단, 푸터를 의미

<header> 태그
1. 머리말 지정
2. 사이트 전체의 헤더는 주로 페이지 맨 위쪽이나 왼쪽에 삽입하며 헤더의 내용으로는 주로 <from>태그를 사용해 검색 창을 넣거나 <nav>태그를 사용해 사이트 메뉴를 넣습니다.

<nav> 태그
1. 문서를 연결하는 네이게이션 링크
2. 네비게이션 역할을 하는 <nav>태그는 동일한 사이트 안의 문서나 다른 사이트의 문서로 연결하는 링크 모음을 나타냅니다,
3. 네비게이션 메뉴뿐 아니라 푸터에 있는 사이트 링크 모음 부분에도 많이 사용됩니다.

<section> 태그
1. 주제별 컨텐츠 영역 나타내기
2. 문맥 흐름중에서 컨텐츠를 주제로 묶을 때 사용하며 그안에는 섹션 제목을 나타내는 <h1>~<h6>제목 태그가 함께 사용됩니다.

<article> 태그
1. 컨텐츠 내용 넣기
2. 웹상의 실제 내용을 넣습니다. 보통 블로그의 포스트나 웹사이트의 내용, 사용자가 등록한 코멘트, 독립적인 웹 컨텐츠 항목을 포함합니다.
3. <article>과 <section>의 차이점은 <section>태그는 문맥 흐름 중에서 컨텐츠를 주제별로 묶을 때 사용합니다.

<aside>태그
1. 본문 이외의 내용 표시
2. 블로그에서 왼쪽이나 오른쪽 또는 하단에 사이드바가 표시된 것과 같은 형태 
3. 필수요속 아니므로 광고나 링크 모음 등 문서의 메인 내용에 영향을 미치지 않는 내용들을 넣을 때 사용


<footer> 태그
1. 제작 정보와 저작권 정보 표시
2. 웹 문서 끝자락에 들어가는 태그
3. <footer>태그 안에는 <header> 태그를 비롯해 <section>,<article>등 다른 레이아웃태그들을 모두 사용할 수 있고, 이런 태그를 이용해 푸터 안에 다양한 저오를 넣을 수 있습니다.

<main>태그 
1. header, footer, nav를 제외한 주요 정보 표시
2. 웹페이지당 하나만 사용

<address> 태그
1. 사이트 제작자 정보, 연라거 정보 나타내기
2. 웹 사이트와 관련된 우편 주소도 태그안에 <address>태그 안에 포함시킵니다.

*검색엔진최적화에서 매우 중요한 요소로 역할을 하기 떄문에 검색결과에 많은 노출을 하고 싶다면 시멘틱 태그를 필수로 사용하는 것이 좋습니다.
