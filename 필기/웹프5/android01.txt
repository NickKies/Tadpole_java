안드로이드란?
1. 모바일 디바이스 플랫폼
2. Application은 JAVA로 작성하고 컴파일은 Dalvik위에서 실행
3. 리눅스 커널을 기반으로 한 모바일 OS
4. 2008년 Alpha , Beta .. 버전 10

안드로이드 아키텍쳐

커널 -> 라이브러리 -> 애플리케이션 프레임워크 -> 어플리케이션

커널
- 안드로이드 플랫폼의 기반
- 리눅스 커널을 사용
- 리눅스 커널을 사용하면 안드로이드의 주요 보안 기능을 활용하고 하드웨어 드라이버를 개발할 수 있음

라이브러리
- core 라이브러리 : Java의 대부분의 기능을 사용
- 달빅VM : Java의 JVM과 같은 역할. 컴파일과 실행을 담당
(달빅VM -> ART)
- C/C++로 구성된 라이브러리

애플리케이션 프레임워크
- 안드로이드에서 제공하는 앱 개발시 도움이 되는 모듈식 시스템 구성요소
- 컴포넌트, 확장 가능한 뷰들
- Resource Manager : 그래픽 및 레이아웃 제공
- Notification Manager : 상태 알림 표시
- Activity Manager : 앱의 수명주기, 서비스등

어플리케이션
- 기본 앱
- 이메일, SMS, 전화, 캘린더 등


액티비티 vs 뷰

액티비티(Activity)
- 안드로이드 프로그램의 화면을 구성하는 단위
- 액티비티가 모여 하나의 프로그램을 구성
- 화면에 직접적으로 보이지 않음

뷰
- 사용자가 실제로 보게되는 화면
- 여러개의 뷰가 모여 하나의 액티비티를 구성

* 레이아웃 : 액티비이 안에 뷰를 배치하는 기법

뷰의 종류
1. 위젯
	- 사용자 인터페이스를 구성(컨트롤)
	- View 클래스로부터 파생되며 스스로 기능을 가지고 있음
2. 뷰 그룹
	- 직접 보이지는 않으며 뷰를 담는 컨테이너 역할
	- 뷰들을 자식으로 포함하여 이를 정렬하는 기능(레이아웃)
