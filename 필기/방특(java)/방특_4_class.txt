 1. 클래스의 구성요소를 쓰고 간단히 설명하세요.
[답변]
 
클래스  : 객체의 속성과 기능을 정의
인스턴스  : 클래스를 통해 만들어진 형태
메서드  : 특정한 작업이나 논리를 구성하는 코드를 괄호로 묶어 놓은 것
오버로딩 : 같은 이름을 가지지만, 매개변수의 개수와 타입이 다른 메소드를 정의
클래스의 상속 : 하위 클래스는 상위 클래스 모두를 적용받습니다.

2. 인스턴스 변수와 클래스 변수를 구분해서 설명하세요.
[답변]
클래스 변수는 프로그램이 시작할 때부터 구동되고 인스턴스 변수는 객체가 있어야합니다.

3. 클래스 이름으로 인스턴스 메소드를 호출할 수 없는 이유를 설명하세요.
[답변]
인스턴스 메소드는 저장공간이 다르기 때문에 아직 생성이 되지 않았을 수 있습니다.

4. 메소드 오버로딩이 필요한 이유를 아는데로 설명하세요.  
[답변]
 
 프로그래밍을 할 때 사용자는 매개변수와 타입이 다른 메소드이지만 같은 기능의 메소드를 
같은 이름으로 정의하고 사용하는 것이 편리하기 때문입니다.

5. 생성자는 new연산자와 같이 쓰는데, 무엇을 하게 되나요?
[답변]
 
 클래스의 객체를 인스턴스가 메모리의 빈 공간에 생성되고 난 다음 
저장된 객체의 메모리 주소 값이 참조변수에 저장합니다.
